About the dashboard


Who are we?

This dashboard has been developed within the EU funded Joint Action project called EU-JAMRAI 2 (2024-2027).

Our goal is to strengthen national AMR surveillance systems in the human sector across participating countries and the European region. This is an active development project which means that alterations will be made continuously. The dashboard is an important tool in our work since the visualization of surveillance information enables joint analysis and discussions.

If you have any input please feel free to reach out, see contact information at the bottom of the page.


What is included in the dashboard?

The dashboard displays the current national microbiological AMR surveillance in the EU-JAMRAI 2 partner countries. The inventory adds to and complements ongoing work at ECDC, WHO and the European Reference Laboratory for AMR. The focus of the inventory is not on AMR data but to collate information of the underlying surveillance structure and systems producing and collecting the data in each country, and how the data is presented and used.


How was the information collected?

The information presented in this dashboard was collected in winter/spring 2025 via a questionnaire developed within the project. You find the questionnaire here.

One coordinator per country was responsible for collecting the answers, filling them in and submitting the questionnaire through the questionnaire tool EUSurvey.


Technical specifications

The dashboard was developed using R Shiny. Technical specifications can be found on our GitHub page.

Calculation of scores

To enable the visualisation of different answeres per country there is a scoring system applied.

Each question has a coefficient (weight) associated to it, which alters how the question should be counted to the overall score. In the same way each possible answer to each question has a value (0-1) associated to it. Multiplying the coefficient (question) with the value (answer) for each question/answer provides a score. A final score between 0 to 1 is obtained by summing all scores, and normalizing using the maximum score. The score is recalculated based on the filters used.
